Ardur is most useful when contributions preserve the evidence discipline:
claims should point to code, specs, tests, media artifacts, or explicit
limitations.

## Useful Contribution Areas

- Lift deferred adapter specs into runnable, tested examples.
- Improve rerunnable proof media and verifier commands.
- Add conformance vectors for public v0.1 specs.
- Harden packaging so Ardur Personal can install without a source checkout.
- Improve deployment walkthroughs without overstating production readiness.

## Before Opening A PR

- Run the local checks in {{< repo-link "docs/TESTING.md" "Testing" >}}.
- Keep generated source mirrors in sync.
- Do not add dead links, fake media cards, or future-tense work as
  present-tense capability.
- Follow {{< repo-link "CONTRIBUTING.md" "Contributing" >}} and the public
  agent instructions.
